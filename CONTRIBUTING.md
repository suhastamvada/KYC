# Contributing

## Vote

Open [Discussions → Proposals](../../discussions/categories/proposals) and
react to any proposal:

- 👍 — I support including this in the draft
- 👎 — I oppose including this in the draft

Reactions are public. One account, one reaction. If you oppose a proposal,
a comment explaining why is far more useful than the reaction alone.

## Propose a reform

1. Search existing proposals first. Duplicates will be closed with a pointer
   to the original.
2. Open a new discussion in the **Proposals** category. The template below is
   required — proposals missing sections will be asked to revise before they
   can be merged.

```markdown
## Problem
What specifically is broken? Scope it tightly.

## Evidence
Sources: reports, data, court records, credible journalism. Claims without
sources will block merging.

## Proposed reform
The concrete change: what rule, structure, process, or resource changes,
and who implements it.

## Tradeoffs and risks
What this costs, who bears it, what could go wrong. A proposal with no
stated tradeoffs is treated as incomplete.

## Affected parties
Students, parents, institutions, administrators, states — who is touched
and how.
```

3. Engage with comments. Revise the proposal body as objections are
   addressed — the discussion's edit history preserves versions.

## Improve the draft directly

Typos, broken links, formatting, and sourcing corrections: open a pull
request against the relevant file in `/draft`. Substantive changes must go
through the proposal process.

## Discuss

Use the **General** category for open-ended discussion and **Meta** for
questions about how this project itself runs.

## Ground rules

See [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md). The short version: argue
about proposals, not people; bring sources; no party-political campaigning.
